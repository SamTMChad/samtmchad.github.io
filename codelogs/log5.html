Images

How Images Are Used in HTML and CSS

Images can be added to a webpage using HTML elements or through CSS styling. In HTML, images are placed directly into the page using the <img> tag. In CSS, images are commonly added as background images using the background-image property. Both methods are important for web design, but they serve different purposes in how images behave, scale, and interact with content.

Difference Between <img> and background-image

The <img> tag is used to place an image directly into the content of a webpage. It is part of the HTML structure and can include alt text for accessibility and SEO. A background image is applied through CSS and is mostly decorative. It does not appear in the page content and cannot include alternative text for screen readers. Images added using <img> affect the layout directly, while background images stay behind other elements and are used mainly for design purposes.

Other CSS Properties That Affect Background Images

Besides background-image, several other CSS properties control how background images appear. The background-size property controls how large the background image appears, such as "cover" or "contain." The background-position property controls where the image is placed within the element, such as center, top, or bottom. The background-repeat property controls whether the image repeats or only appears once.

Dimensions vs Resolution (One-Sentence Definition)

Dimensions refer to the width and height of an image in pixels, while resolution refers to how detailed the image is based on pixel density.

Extra Credit: Image File Formats

GIF, PNG, JPG/JPEG, WebP, and SVG are all common image formats used on the web, but they are designed for different purposes. GIF supports simple animations and has limited colors, which makes it good for small animated graphics. PNG supports transparency and high image quality, making it great for logos and graphics with sharp edges. JPG/JPEG compresses images very efficiently, making it ideal for photographs but it does not support transparency. WebP is a modern format that supports both transparency and compression, often producing smaller file sizes than JPG and PNG. SVG is a vector format, meaning it is made of mathematical paths instead of pixels, allowing it to scale perfectly at any size without losing quality.
